# 🎮 Unity TPS Player System (Submission)

> TPS 전투 시스템 확장을 고려한 플레이어 컨트롤 구조입니다.

Unity 기반 TPS(3인칭) 플레이어 시스템을 모듈화 구조로 설계 및 구현한 제출용 코드입니다.  
입력, 이동, 조준, 애니메이션을 컴포넌트 단위로 분리하여 유지보수성과 확장성을 고려했습니다.

---

## 📌 주요 기능

- TPS 캐릭터 이동 및 회전 (카메라 기준 이동)
- 락온 / 자유 조준 시스템 분리
- 점프 및 회피(Dodge) 시스템
- 상태 기반 이동 제어 (Jump, Dodge 등)
- Animator 연동 애니메이션 제어
- 입력 시스템 추상화 (인터페이스 기반)

---

## 🧩 시스템 구조
Player (컨트롤러)

├ PlayerMotor (이동, 점프, 회피)

├ PlayerInput (입력 처리)

├ PlayerFreeAim (자유 시점 조준)

├ PlayerLockOnAim (락온 조준)

└ PlayerAnimBridge (애니메이션 처리)



---

## ⚙️ 핵심 설계 포인트

### 1. 컴포넌트 기반 구조
각 기능을 독립적인 컴포넌트로 분리하여 결합도를 낮추고 유지보수를 용이하게 설계했습니다.

### 2. 입력 추상화
`IPlayerInput` 인터페이스를 통해 입력 시스템을 분리하여 확장성과 테스트 용이성을 확보했습니다.

### 3. 상태 기반 제어
Jump, Dodge 상태를 분리하여 이동 및 입력 충돌을 방지했습니다.

### 4. 락온 / 자유 이동 분리
TPS 전투 흐름을 고려하여 락온 상태와 일반 이동 상태를 명확히 분리했습니다.

---

## 📂 주요 코드 설명

- **Player.cs**  
  → 전체 흐름 제어 (입력 → 이동 → 조준 → 상태 처리)

- **PlayerMotor.cs**  
  → 이동, 점프, 회피, 락온 상태 관리 (핵심 로직)

- **PlayerInput.cs**  
  → 입력 처리 및 인터페이스 구현

- **PlayerFreeAim / PlayerLockOnAim.cs**  
  → 조준 방식 분리 (자유 / 락온)

- **PlayerAnimBridge.cs**  
  → 애니메이션 제어 분리

---

## 🚀 개발 의도

TPS 게임에서 중요한 **조작감과 전투 흐름**을 고려하여  
입력 → 이동 → 조준 → 애니메이션이 자연스럽게 연결되도록 설계했습니다.

또한 기능별 분리를 통해 이후 전투 시스템, 무기 시스템 등으로  
확장 가능한 구조를 만드는 것을 목표로 했습니다.
